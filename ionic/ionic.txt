- ionic start projectName flag
	flags:
		- tabs: простая схема с тремя вкладками
		- sidemenu: макет с настраиваемым меню сбоку
		- blank: голой стартер с одной страницей
		- super: стартовый проект с более чем 14 готовыми к использованию дизайна страниц
		- tutorial: проект с управляемым стартером

- cd projectName

- ionic serve

Структура проекта:
	- ./src/index.html 
		- Точка входа. Для работы вашего приложения Ionic ищет <ion-app> тег в вашем HTML.

	- build/main.js
		- содержит Ionic, Angular и JavaScript

	- cordova.js будет не доступен при локальной разработке

	- код внутри директории src будет компилиться в валидный js

	- src/app/app.module.ts
		- вторая точка входа приложения

		- здесь объявляем существующие модули

	- каждое приложение имеет корневой модуль

	- src/app/app.component.ts
		- это первый компонент, который мы загружаем и как правило это пустая оболчка для загрузки других компонентов

	- ./src/app/app.html
		- основной шаблон приложения

- Приложение сделанно из компонентов

- Работа с http:
	- документация в разделе Native
	- ionic cordova plugin add cordova-plugin-advanced-http
	- npm install --save @ionic-native/http
	- добавить данный плагин в app.module.ts, раздел providers

Тестирование в браузере в виде мобильного приложения:
	- ionic serve --lab

- Создание провайдера:
	- ionic g provider Rest (как я понял встроенный провайдер)
	- появляется папка providers и в ней наш rest provider, также данный провайдер подключается в app.module.ts автоматически

- Promise:
	- Promise – это специальный объект, который содержит своё состояние. Внутри промиса можно делать любые асинхронные операции, а когда они завершаться вызвать функцию resolve(res) в случае успеха или reject(err)

- Menu:
	- чтобы добавить меню к приложению элемент <ion-menu> должен быть добавлен к элементу <ion-nav>

	- для открытия/закрытия меню нужно использовать директивы MenuToggle, MenuClose

	- для управления меню используем MenuController

	- для управления навигацией из корневого компонента нужно добавить переменную в ion-nav и использовать @ViewChild
